# 2026-06-17 更新记录

## 长线策略清理

- 智能推荐页移除“长线”策略入口，当前页面策略保留短线、激进突破型、多因子稳健型。
- T+1 默认预热策略从 `短线,长线,多因子稳健型,激进突破型` 调整为 `短线,多因子稳健型,激进突破型`。
- 调度预热目标中只有短线按四个板块生成；激进突破型和多因子稳健型仍只生成 `全部`。
- `RecommendationService` 不再接受长线策略分发；传入 `长线` 或未知策略会明确报错，避免旧 session 或外部调用误触发长线推荐。
- 底层长线推荐实现、长线评分权重、本地 `.env` 旧长线预热配置和本地 T+1 长线缓存/历史记录已清理。

## 短线闭环学习实验版

- 新增 `short_term_learning.py`，只读取已保存短线 T+1 计划和真实后续 K 线回看结果，生成短线学习校准；不使用模拟数据。
- 短线学习层额外防止科创板代码误入，动态评分门槛不写死为 `65`，而是从真实历史回测分数段中选择；完成样本不足时保持观察状态，不启用动态门槛。
- 短线结果会补充 `learning_bonus`、`learned_alpha_score`、`learning_reason` 等字段，用于实验验证学习可信度；激进突破型和多因子稳健型不接入学习层，规则保持不变。

## 短线闭环学习后续阶段

- 阶段 1：真实样本积累。短线 T+1 已保存计划需要积累足够完成样本；当前实验门槛为至少 `12` 个真实完成样本，样本不足时只观察，不启用动态门槛。
- 阶段 2：动态评分验证。样本足够后，检查真实回测分数段是否能稳定区分好坏；只有胜率、1 日均收益优于基准的分数段，才作为动态评分门槛和学习加权依据。
- 阶段 3：可信度复盘。对比原短线推荐和学习后短线推荐的胜率、平均收益、亏损标的占比；连续多轮有效后，再评估是否扩大到项目其他评分模块。
- 后续判断原则：在真实样本足够并且分数段表现稳定前，不把项目评级/评分体系整体切换为回测数据驱动；当数据足够且结果显示更可靠时，需要提醒用户可以进入下一阶段或评估是否更改项目评分依据。
