# Bug fixes & context 2026-06-08

## 决策委员会看多/看空证据归集修复

- 修复个股分析页 A 股决策委员会“看多依据”中可能混入负向技术证据的问题。旧逻辑按 Agent 总分正负取证据，某个 Agent 总体偏多时，前两条 evidence 里如果包含“空头趋势”“超买”等负向信号，也会被展示到看多依据。
- 现在 `decision_committee.py` 在汇总 `bullish_points` / `bearish_points` 时按单条证据方向再次过滤：`空头`、`死叉`、`超买`、`回调`、`偏空`、`下方`、`流出` 等进入看空因素；`多头`、`金叉`、`超卖`、`反弹`、`偏多`、`上方`、`流入` 等进入看多依据；`中性` 或暂无类文本不再强行归入看多。
- 指标语义保持不变：RSI `>70` 为超买、`<30` 为超卖；KDJ 使用 K 值 `>80` / `<20` 判断 K 值超买/超卖。该状态是具体指标自己的阈值状态，不代表全部指标同时超买或超卖。
- 本次未改变 A 股决策委员会评分、置信度、动作、仓位、推荐池、T+1 计划或技术指标计算公式，只修复看多/看空展示归集口径。

## 扩展信息分层缓存部分命中修复

- 修复 `StockInfoService` 读取扩展信息分层缓存时必须同时命中财务摘要和资金流两层的问题。旧逻辑中，资金流短缓存过期或为空时，即使财务摘要长缓存仍有效，也会整包放弃分层缓存并重新请求外部 provider。
- 现在财务摘要和资金流任意一层可用即可返回 `layered_cache`：可用层保留真实缓存数据，缺失层返回 `{}`，避免因为资金流接口短时不可用或缓存过期而拖累财务信息展示。
- 本次不改变真实数据优先级、不引入模拟资金流或假财务数据；缺失层继续按空结构展示，由页面数据质量说明表达缺口。
- 同步补充 `tests/test_data_services.py` 回归用例，覆盖“财务缓存有效、资金流缓存过期”时不调用外部 provider 的场景。
