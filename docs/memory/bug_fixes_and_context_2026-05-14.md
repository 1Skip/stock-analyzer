# 2026-05-14 更新记录

## 背景

今天主要围绕“让项目更像一个可长期使用的 A 股决策工作台”推进：一方面继续借鉴 `TradingAgents`、`daily_stock_analysis`、`a-stock-data` 的架构与能力，另一方面集中修复 Web 端交互、搜索、自选股、推送配置和页面加载体验中的实际问题。

## 今日需求落地

### 1. A 股决策委员会与 TradingAgents 适配

- 借鉴 `TradingAgents` 的多角色框架，但按 A 股场景落地为本项目的五层 Agent：技术分析、资金情绪、基本面、题材板块、风险事件。
- 个股分析页展示决策仪表盘：综合评分、置信度、建议仓位、关键价位、买卖点、风险警报、催化因素和 Agent 折叠明细。
- 日报从“摘要”升级为“决策仪表盘”，输出评分、买卖点、风险警报、催化因素、操作检查清单。
- 可选接入外部 LLM 多空辩论：通过 `AI_DEBATE_ENABLED=true` + `AI_API_KEY` 启用；未配置时自动跳过，不影响普通分析和飞书推送。
- 侧边栏新增决策委员会状态卡片，让网页端能直接看到阶段 1-5、飞书、Actions、LLM 辩论是否启用。

### 2. 每日推送、飞书与 GitHub Actions

- GitHub Actions 工作流支持工作日北京时间 15:30 自动运行，电脑关机也能推送。
- 推送顺序固定为：自选股摘要 → 四板块推荐 → 每日完整 Markdown 日报。
- 固定四板块推荐：算力租赁、电力、苹果概念、特斯拉概念。
- 推荐股推送不再使用全市场推荐股做补充内容，减少噪声和耗时。
- 智能推荐 / 推荐股推送仅推荐沪深主板股票；热门板块、行业排行、概念排行、涨跌幅榜保留全市场。
- 支持飞书和企业微信 Webhook，配置页补充企业微信教程，一键测试推送显示中文通道名。
- 云端自选股支持 `STOCK_LIST` / `WATCHLIST_JSON` Secret，`STOCK_LIST` 支持 A 股代码和中文名称。
- 推荐股推送补齐“交易计划卡片 + 风控防御看板 + 资金博弈溯源”，四板块推荐和自选股推送口径保持一致。
- 配置页明确“一键测试推送”只验证 Webhook 连通性，不生成正式股票分析内容。

### 3. 自选股与中文股票名称识别

- A 股中文名称搜索补强：使用全量 A 股名称索引 + 本地缓存 + 静态兜底，避免“说一个补一个”。
- 支持中文名称错序/模糊匹配，例如“顺捷科技”可识别到“捷顺科技”。
- 个股分析页恢复“加入自选”入口，并保证自选股持久化保存到 `watchlist.json`。
- 自选股侧栏改为常驻，不依赖是否已搜索股票；侧栏加载更轻量。
- 点击自选股后，主页直接显示所点击股票的个股分析数据。
- 移除重复的“自选详情”面板，避免和主页完整分析入口重复。
- 自选股列表显示股票名称 + 代码，而不是只显示代码。

### 4. 个股分析、新闻与基础资料

- 个股分析输入框顶部新增“股票代码或名称”说明，明确支持代码和中文名称。
- 修复首次搜索时“财务 / 资金 / 新闻”折叠栏不出现，第二次搜索才出现的问题。
- 新闻层补充东方财富个股新闻、财新数据通市场资讯，并修复新闻为空的展示逻辑。
- 恢复“基础资料 / 估值”折叠栏，避免修复新闻时误删原有功能。
- A 股行业、上市日期、股本、市值、PB 等字段增加全量基础资料索引兜底。
- 行业字段尽量对齐同花顺公司概况口径，避免只显示粗粒度行业代码。
- AI 辅助解读保留为可选解释层，主结论以 A 股决策委员会为准。

### 5. 股票对比与回测体验

- 股票对比支持输入股票代码或中文名称，和个股分析保持一致。
- 价格走势对比升级为走势决策仪表盘，补充区间收益、最大回撤、波动率、上涨天数占比、MA 状态、趋势斜率和相对强弱。
- 回测页面新增“股票代码或名称”标题栏，避免只显示代码造成理解成本。
- 回测输入框支持按 Enter 触发分析。
- 修复部分新股/短历史股票显示“数据不足”时名称展示不完整的问题。

### 6. UI、加载和页面切换体验

- 热门板块页改为点击后再获取数据，避免进入其他页面时仍被热门板块请求拖慢。
- 智能推荐切换策略/板块/数量会清理旧结果，避免旧推荐残留。
- 页面切换时清理非当前页面的展示状态，减少从智能推荐切到股票对比时旧页面残留。
- 自定义加载状态替换多处传统转圈，减少“页面空白等待”的感觉。
- 个股分析搜索中保留页面结构并显示进度，不再长期空白。
- 指标图补充实时数值，统一 MACD/RSI/KDJ/BOLL 折叠与标识位置，修复 BOLL/价格遮挡和 MACD 数值缺失。
- 大盘温度提前到侧栏同步渲染，并发获取上证、深证、沪深300、北证50；优先走新浪快速源，单次实测约 0.257s。
- Web 全局主题改为黑蓝科技风，主按钮统一为卡里布蓝 `#015697`，避免黄色按钮与白色文字冲突。
- Tab 选中态改为白色文字，并移除额外胶囊背景，只保留清晰的选中提示。
- 项目协作约束写入 `CLAUDE.md`：没有用户明确要求，不擅自新增 UI、背景装饰、交互形态或额外功能。

### 7. 本机启动与项目分发

- `start.bat` 作为 Windows 一键启动入口：自动创建 `.venv`、安装依赖、启动 Streamlit 并打开浏览器。
- `创建桌面快捷方式.vbs` 用于生成桌面快捷方式，指向项目的一键启动脚本。
- `install_startup.bat` / `uninstall_startup.bat` 用于设置或取消 Windows 登录后自动启动。
- 删除无用/重复启动脚本，保留清晰的启动路径。
- `.env.example` 保留公开模板；`.env` 为本机私密配置，不提交 Git。

## 注意事项

- `.env` 和 `watchlist.json` 是本机私密/运行态文件，不提交 GitHub。
- GitHub Actions 使用的自选股不会自动读取本机网页端 `watchlist.json`；云端需要配置 `STOCK_LIST` 或 `WATCHLIST_JSON` Secret。
- 飞书 Webhook 主动推送不是实时对话 Agent；实时对话仍需要 FastAPI 回调服务、公网访问和飞书事件订阅。
- 热门板块保留全市场，智能推荐和推荐股推送才做沪深主板过滤，这是当前明确规则。
- GitHub Actions 中 AKShare/东方财富接口偶发 `RemoteDisconnected`、`TimeoutError` 多为云端网络或数据源限流，不等同于 Webhook 配置错误；应优先用股票代码配置 `STOCK_LIST`，减少中文名实时解析依赖。
- 后续修复遵守“只改用户明确要求范围”的协作约束，附带调整必须先说明原因。
