# Bug fixes & context 2026-05-19

## 今日已完成

- 修复 A 股个股分析页指标与同花顺口径不一致问题：A 股个股分析固定使用 `1y` 前复权日 K，实时行情不再合并进原始日 K 参与 MACD/KDJ/BOLL/MA 计算。
- 个股分析页均线展示从 MA60 改为 MA30，展示口径为 MA5/MA10/MA20/MA30，所有指标展示值保留两位小数。
- K 线图与成交量图同步使用同一份前复权日 K；成交量按万手展示，并补充成交量 MA5/MA10。
- 智能推荐页展示指标在选股完成后重新按个股分析页同一口径计算，推荐页展示的 MACD/KDJ/BOLL/MA 与个股分析页保持一致；本次没有修改 `stock_recommendation.py` 的选股策略和评分规则。
- `CLAUDE.md` 已写入红线：个股分析页 A 股技术指标、K 线和成交量图口径不得擅自修改；实时行情只用于最新价展示、分时或入场执行检查，不得合并进个股分析页指标计算；智能推荐展示指标必须与个股分析页复用同一口径。
- README 已明确项目定位：收盘后/开盘前选股 + 盘中执行检查，不作为盘中实时全市场重新选股或高频交易系统。
- 智能推荐页补充展示基础资料：行业、市值、PE、PB、换手率和来源；真实接口未返回时明确显示暂无，不使用假数据补齐。
- 推荐解释里的 `extended_info` / `profile` 内部字段改为中文证据说明，避免把技术字段直接暴露给用户。
- 热门板块页的行业/概念排行空数据提示改为明确说明“数据源未返回有效结果”，个股涨跌幅榜仍可正常参考。
- 推荐页不再展示 `not_requested`、生成耗时、预热状态等内部调试信息；无效实时行情报价（价格 0 或 -100%）不会覆盖推荐原始价格。

- 金富科技 `003018` 只是验证样例，不是单股特例修复；修复落在 A 股个股分析数据获取、共享指标上下文和智能推荐展示指标刷新链路，对所有 A 股生效。
- 选股策略不改变，不牺牲准确率：本次只统一展示指标/K线口径，不调整原始选股条件、推荐排序和评分规则。
