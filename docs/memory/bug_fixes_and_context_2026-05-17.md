# Bug fixes & context 2026-05-17

## 今日已完成

- 智能推荐新增 T+1 推荐计划闭环：`run_t1_plan()` 负责生成计划，`latest_t1_plan()` 负责读取同条件缓存，`check_entry_plan()` 只对计划内股票做次日入场检查。
- T+1 计划缓存按“策略 + 板块 + 推荐数量”隔离，缓存命中时不重新扫描三千多只股票；页面展示生成耗时、缓存读取耗时、缓存状态和预热状态。
- `scheduler.py` 接入 T+1 计划预热入口，默认关闭，可通过 `T1_PLAN_AUTO_ENABLED` 等配置在收盘后提前生成计划。
- 入场检查实时行情链路为东方财富 → 新浪 → 腾讯；实时行情只用于入场检查，不参与推荐选股、不改变推荐排序、不新增或删除推荐股票。
- 入场检查增强了 ST/退市风险、公告风险、疑似停牌、高开偏离、跌破 BOLL 下轨等执行层判断；这些规则只影响“是否适合入场”的提示，不影响选股条件。
- 智能推荐页增加请求 key 与运行状态隔离：生成中不渲染旧推荐、旧诊断、旧入场检查；切换策略/板块/数量后不显示上一轮结果；不同策略的运行状态不再混淆。
- 个股分析、股票对比、热门板块、智能推荐等页面接入真实阶段进度条/进度提示。
- 个股分析页修复连续搜索时旧结果残留问题：输入新股票后会清空上一只股票结果并按当前查询展示。
- AI 辅助解读区域修复生成过程中旧结果/旧错误残留，运行中禁用按钮并在完成后刷新显示稳定结果。
- 个股决策仪表盘 PEG 优先复用 `astock-peg` 思路：forward PE = 当前价 / 下一年度一致预期 EPS，PEG = forward PE / 利润 CAGR；数据不足时回退到已有 PEG/PE 与增长率口径。
- `AkShareInfoProvider` 财务归一化保留利润历史，用于 PEG 的利润 CAGR 计算。
- pytest 环境跳过本地 `.env` 注入，避免本机密钥影响默认配置测试；应用正常运行仍读取 `.env`。
- 生成并保留 AI 编码协作 12 条执行规则文档：`docs/generated/AI编码协作12条执行规则.docx` 与 `.pdf`。
