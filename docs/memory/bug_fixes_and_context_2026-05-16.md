## 今日需求落地

### 1. 智能推荐策略定版

- 新增策略选择：`综合推荐`、`激进突破型`、`多因子稳健型`。
- 推荐范围：沪深主板 + 创业板。
- 排除范围：科创板、北交所、ST、退市/异常股票。
- 数据原则：只使用真实数据源；缺失显示“缺失/接口失败/未通过”，不使用模拟数据。

### 2. 激进突破型

- 定位：短线/超短线，纯量价突破，不看基本面。
- 硬性范围：沪深主板 + 创业板，排除科创板、北交所、ST、退市/异常股票。
- 市值条件：总市值 `<300亿`。
- 技术条件：`MA5 > MA10 > MA20`。
- 突破条件：最新收盘价创近 `20日` 新高。
- 量能条件：当日成交量 > 前 `5日` 平均成交量 × `1.2`。
- 买入提示：突破确认后观察买入，不承诺实时打板能力。
- 卖出纪律：次日低开超过 `-5%` 止损；跌破前收且 `5` 分钟不反弹退出；高开冲高分批止盈；平开 `30` 分钟走弱退出。

### 3. 多因子稳健型

- 适用范围：沪深主板 + 创业板。
- 排除范围：科创板、北交所、ST、退市/异常股票。
- 硬性过滤：总市值 `<300亿`。
- 硬性过滤：近 `10日` 涨幅不超过 `35%`。
- 硬性过滤：无重大风险事件。
- 核心因子：均线金叉或均线多头，并且放量。
- 核心因子：最新季报净利润同比 `>20%`，或者最新季报未亏损。
- 核心因子：连续上涨 `3日`。
- 核心因子：主力净流入资金趋势 `>=3000万`。
- 核心因子：既往 `15日` 内有涨停。
- 入选规则：核心因子至少 `3/5` 通过，且综合评分 `>=70`。
- 重要口径：`既往15日内有涨停` 指盘中最高价触及涨停即可，不要求收盘涨停。

### 4. 查询速度优化

- 历史 K 线：本地缓存，按交易日更新一次。
- 当日价格/成交量：运行推荐时实时拉取一次，并合并到最后一根 K 线。
- 市值/行业/上市日期：本地缓存，按慢变数据处理。
- 财务/研报/公告：按天缓存。
- 资金流：短时缓存，适合 30-60 分钟级别复用。
- 页面提供“刷新数据”按钮，用于手动刷新策略 K 线缓存。
- 快速模式使用本地缓存 + 当日实时补偿，避免每次网页点击都给 4000+ 股票重新拉取 3 个月 K 线。

### 5. 真实阶段进度

- 激进突破型进度：股票池 → 当日实时价量 → K线轻筛 → 市值过滤 → 完成。
- 多因子稳健型进度：股票池 → 市值过滤 → 当日实时价量 → K线轻筛 → 深度检查 → 完成。
- 进度条展示真实阶段和计数，避免只显示“运行 0 秒”或不可解释的空转状态。

### 6. 推送同步

- 飞书/企业微信推荐股推送同步支持激进突破型候选和多因子稳健型候选。
- 推送内容包含买入观察点、卖出纪律、风险提示。
- 自选股摘要、推荐股推送和日报继续同步交易计划卡片、风控防御看板。

### 7. 推荐服务层、缓存拆分与页面复用

- 新增 `recommendation_service.py`，网页智能推荐和固定四板块推送共用同一套推荐分发入口；服务层只负责路由策略、刷新最终实时价、持久化结果，不改变 `StockRecommender` 内的选股条件和评分规则。
- 智能推荐页面改为调用 `RecommendationService`；相同“策略 + 板块 + 数量”的最近结果会写入 `recommendation_results` 缓存，页面刷新或切页回来可直接展示，避免短时间内重复全量扫描。
- 扩展信息缓存拆分为财务、资金流、研报/分红/板块归因、公告/风险事件等独立缓存，并保留旧的整包扩展信息缓存兼容路径；资金流默认 3600 秒，财务/研报/风险事件默认 86400 秒。
- `JsonFileCache` 增加跨进程写锁，降低多因子深度检查子进程并发写缓存时互相覆盖、导致缓存条目缩水的问题。
- 多因子深度检查进度新增 `deep_total` / `deep_done` / `result_count` 逐批更新，页面不再长时间只停在“深度检查 N”；本次没有截断候选，也没有降低全市场覆盖率。
