# 股票分析系统

个人股票技术分析工具，支持 A股/港股/美股，提供 Web 界面和 CLI 两种使用方式。

## 项目定位与使用边界

本项目的核心定位是 **收盘后/开盘前选股 + 盘中执行检查**，不是盘中实时重新选股或高频交易系统。

- **适合使用的场景**：收盘后复盘、生成次日 T+1 候选计划、开盘前查看已生成推荐、盘中对计划内股票做入场/风控检查。
- **不适合作为实时选股依据**：不用于盘中实时扫描全市场重新选股，不根据实时价格不断新增、删除或重排原始推荐列表，不用于打板抢单、秒级交易或高频决策。
- **实时行情的边界**：实时行情只用于最新价展示、分时图、自选股摘要和计划内股票的盘中执行检查；不合并进原始日 K 线参与个股分析页指标计算，也不参与原始选股条件。
- **核心选股口径**：智能推荐的原始选股以稳定的历史日 K、收盘后数据和公开基础/资金/风险数据为主，优先保证策略口径一致和结果可复核。

## 项目协作规则

- **红线来源** — `CLAUDE.md` 是项目红线、架构约定和长期协作规则的正式来源，包含需求执行、需求记录、个股分析指标口径、真实数据和安全边界。
- **Codex Skill** — `.codex/skills/stock-analyzer/SKILL.md` 是随仓库提交的 Codex 执行 SOP，用于提醒 AI 在本项目中先查红线、按真实链路验证、区分缓存/推送/调度状态，并执行“更新推送”固定流程。
- **分工关系** — Skill 不替代 `CLAUDE.md`；`CLAUDE.md` 记录项目规则，Skill 把规则压缩成 Codex 工作时的检查清单。

## 2026-05-17 推荐缓存与页面状态更新

- **Alpha 推荐评分** — 智能推荐新增确定性 Alpha 二次评分，按策略原始分、趋势、量能、资金、板块、财务、风险和追高风险输出 `0-100` 分、等级、排序理由和扣分项；默认只展示评分，不改变原策略选股与排序，可通过 `RECOMMEND_RANKER_SORT=true` 开启按 Alpha 分重排。
- **T+1 推荐计划** — 智能推荐页新增“生成 T+1 推荐计划”和同条件计划缓存读取；相同“策略 + 板块 + 推荐数量”命中缓存时直接展示昨晚/上次生成结果，不重新扫描三千多只股票。
- **推荐买卖点** — 推荐计划生成后会追加 `trade_plan`，展示买入观察区、加仓确认、止损线、压力/止盈位、失效条件和仓位建议；买卖点只读取收盘后的日 K 技术位，不参与选股、评分或排序。
- **准确率与选股边界** — T+1 计划、缓存读取、页面状态隔离和入场检查均不修改“激进突破型 / 多因子稳健型”选股条件；实时行情只用于次日入场检查，不参与选股、不重排、不新增、不删除推荐列表。
- **入场检查兜底** — 对 T+1 计划内股票按东方财富 → 新浪 → 腾讯顺序获取实时行情；全部失败时明确提示实时行情不可用，不生成假入场建议。
- **定时预热** — `scheduler.py` 已接入 T+1 计划预热入口，默认开启；默认每天 `15:45` 按“短线/长线 × 全部/苹果概念/特斯拉概念/电力/算力租赁 + 多因子稳健型/激进突破型 × 全部 / 5只”分别提前生成计划，可通过 `T1_PLAN_AUTO_ENABLED`、`T1_PLAN_SCHEDULE_TIME`、`T1_PLAN_STRATEGIES`、`T1_PLAN_SECTORS`、`T1_PLAN_NUM_STOCKS`、`T1_PLAN_PREHEAT_KLINE`、`T1_PLAN_PREHEAT_EXTENDED_INFO` 调整。
- **T+1 推送** — 调度预生成完成后可把已生成计划推送到飞书/企业微信，推送正文包含推荐股交易计划卡片和风控防御看板，不重新选股。
- **自选股同步 Actions** — 网页端 `watchlist.json` 可通过 `github_watchlist_sync.py` 同步到 GitHub Actions 的 `WATCHLIST_JSON` Secret，云端日报和自选股摘要会按同步后的自选股执行。
- **页面残留修复** — 智能推荐页按请求 key 隔离运行状态，生成中不展示旧推荐、旧诊断或旧入场检查；不同策略各自显示状态，避免稳健型和激进型互相混淆。
- **其他页面进度** — 个股分析、股票对比、热门板块、智能推荐等页面接入真实阶段进度提示，优先展示当前执行阶段而不是无意义等待。
- **个股分析修复** — 连续搜索不同股票时会清空旧结果并按当前输入展示；切换到其他页面再回到个股分析时，顶部搜索框、当前标的和下方分析数据会统一到同一个已分析标的，避免旧输入和新数据错位。
- **PEG 口径** — 个股决策仪表盘 PEG 优先复用 `astock-peg` 思路：用当前价 / 下一年度一致预期 EPS 得到 forward PE，再除以利润 CAGR；一致预期或历史利润不足时再回退到已有 PEG/PE 与增长率口径。
- **资金流边界** — 公共资金流接口不可用时继续显示缺失或接口失败，不使用模拟值、随机值或手写假数据补齐“主力净流入趋势”。
- **深度进度修复** — 多因子稳健型进入深度检查时会显示深度总数、已深查数量和当前命中数；初始阶段从 `已深查 0 / 最终命中 0` 开始，后续按批次持续更新。
- **验证结果** — 本轮完整测试已通过：`776 passed, 20 warnings`。

## 2026-05-16 智能推荐策略更新

- **推荐范围** — 智能推荐新增支持“沪深主板 + 创业板”，仍排除科创板、北交所、ST、退市风险和明显异常停牌股票；热门板块排行继续保留全市场口径。
- **激进突破型** — 纯量价策略，要求总市值 `<300亿`、`MA5 > MA10 > MA20`、最新收盘价创近 `20日` 新高、当日成交量大于前 `5日` 均量 `1.2` 倍；页面和推送都会展示买入观察与隔日超短卖出纪律，次日低开止损线为 `-5%`。
- **多因子稳健型** — 先硬性过滤总市值 `<300亿`、近 `10日` 涨幅不超过 `35%`、无重大风险事件；再按“均线金叉或均线多头并放量、最新季报净利润同比 `>20%` 或未亏损、连续上涨 `3日`、主力净流入资金趋势 `>=3000万`、既往 `15日` 内有涨停”五个核心因子评分，核心因子至少 `3/5` 且综合评分 `>=70` 才进入候选。
- **速度优化** — 历史 K 线使用本地交易日缓存，资金流短时缓存，财务/公告/研报按天缓存；页面提供“刷新K线缓存”按钮用于手动更新推荐策略日 K 缓存，不是读取或生成 T+1 推荐计划。实时行情只用于计划内入场检查和最新价展示，不参与原始选股。
- **推荐服务层** — 网页智能推荐、固定四板块推送共用 `RecommendationService` 分发短线/长线/激进突破型/多因子稳健型；相同策略/板块/数量的网页结果会在短时缓存内复用，避免刷新或切页后重复跑。
- **缓存拆分与进度** — 扩展信息拆为财务、资金流、研报/板块、风险公告等缓存，并加跨进程写锁；多因子深度检查会持续显示已深查数量、总数和当前命中数，选股条件与评分规则不变。
- **推送同步** — 四板块推荐推送除原“短线/长线”外，新增“激进突破型/多因子稳健型”候选；所有策略只使用真实行情、资金、财务、风险事件等公开数据，缺失项会标注为缺失/接口失败/未通过。
- **主力吸货指标** — 个股分析页和推送新增同花顺公式版“主力吸货 / 风险 / 涨跌”，完全由真实日 K 的 high/low/close 推导；这是公式指标复刻，不读取同花顺插件私有数据，不参与选股、评分或排序。
- **文档同步约定** — 每次需求改动或 bug 修复完成后同步记录到 memory；README 只记录用户明确提出、已经确认或已经完成验证的功能与修复，不把未确认的扩展想法写成正式需求。

## 功能特点

### 技术指标
- **K线图** — Plotly 交互式蜡烛图（Web），Matplotlib 静态图（CLI）
- **MACD** — 趋势判断和买卖时机，含金叉/死叉标注
- **RSI** — 6/12/24 三周期相对强弱指数
- **KDJ** — 随机指标，含金叉/死叉标注
- **BOLL** — 布林带，压力位/支撑位
- **MA** — 个股分析页展示 5/10/20/30 四条均线，指标层仍保留 MA60 供长期风控模块使用
- **主力吸货** — 按同花顺公式系统实现 `主力吸货`、`风险`、`涨跌` 三条输出，基于真实前复权日 K 推导，仅用于展示、解释和推送，不改变推荐策略
- **分时图** — 当日价格走势 + 均价线 + 昨收参考线

### 当前推荐与推送规则

- 智能推荐主要用于收盘后/开盘前生成候选股和 T+1 计划；盘中只对计划内股票做执行检查，不做全市场实时重新选股。
- 热门板块页用于观察市场热度：行业板块排行、概念板块排行、个股涨幅榜/跌幅榜均保留全市场，不做主板过滤。
- 智能推荐和定时推荐股推送用于辅助决策：短线/长线策略保留原有沪深主板口径；激进突破型/多因子稳健型扫描沪深主板 + 创业板，科创板、北交所、ST、退市/异常股票不进入推荐池。
- 实时行情只用于最新价展示、分时图、自选股摘要和 T+1 计划内股票的入场检查；不参与原始选股、不改变排序、不新增或删除推荐股票。
- 智能推荐页面会复用最近一次相同“策略 + 板块 + 数量”的推荐结果缓存；首次全市场多因子查询仍按完整条件扫描，不截断候选，不降低覆盖率。
- 智能推荐页面的买卖点在推荐计划生成后追加，使用收盘后的日 K、BOLL、均线、支撑/压力和风险字段生成；不做盘中实时追高、跌破支撑或暂停入场重算。
- 定时推送顺序为：自选股摘要 → 四板块推荐 → 每日完整 Markdown 日报；四板块固定为算力租赁、电力、苹果概念、特斯拉概念。
- 四板块推荐默认每个板块短线 2 只 + 长线 1 只，并同步输出激进突破型/多因子稳健型候选；短线/长线数量可通过 `SECTOR_PUSH_SHORT_TOP_N` / `SECTOR_PUSH_LONG_TOP_N` 调整。
- 自选股摘要、四板块推荐股和每日完整日报都会同步推送“交易计划卡片 + 风控防御看板 + 资金博弈溯源”；自选股摘要、四板块推荐和 T+1 推荐计划还会展示主力吸货公式指标，与网页端个股分析口径保持一致。
- GitHub Actions 自选股优先读取 `WATCHLIST_JSON` Secret；本地网页自选股同步功能会覆盖该 Secret，使云端推送跟随当前网页自选列表。
- 配置页“一键测试推送”只验证飞书/企业微信 Webhook 是否打通，不会生成正式股票分析内容。

### 交易计划与风控防御看板

个股分析页已把原有“买卖点与仓位”升级成更明确的单股交易计划模块：

- **交易计划卡片**：回答“当前能不能买、买在哪里、错了在哪里止损、涨到哪里先减仓”。
- **交易状态机**：把单股状态明确落到“防御观望 / 底部观察 / 试探建仓 / 趋势持有 / 压力减仓 / 风险回避”，并给出下一步触发条件。
- **风控防御看板**：回答“这只股票现在是安全低吸、趋势持有、追高风险，还是应该回避”。
- **资金博弈溯源**：展示当日主力、近 5 日主力、超大/大单、主力净占比、换手率、20 日量价成本等字段，区分公开数据与模型推断。

计划展示字段：

| 模块 | 展示内容 |
|------|----------|
| 当前动作 | 建仓 / 轻仓试探 / 持有 / 等待确认 / 降仓回避 |
| 买入观察区 | 接近支撑位、布林下轨、MA20/MA60 或放量突破确认位 |
| 加仓确认位 | 重新站上关键均线、突破压力位、趋势和资金共振 |
| 止损线 | 跌破支撑、跌破布林下轨、跌破 MA60 或风险事件触发 |
| 止盈/减仓位 | 接近压力位、偏离均线过大、RSI 超买或放量滞涨 |
| 仓位建议 | 观察仓 / 1-2 成 / 2-3 成 / 降仓 / 空仓 |
| 风控评分 | 估值、成长、趋势、安全、资金五维评分和解释 |
| 状态机 | 防御观望、底部观察、试探建仓、趋势持有、压力减仓、风险回避 |
| 核心指标 | PEG、相对强弱、Beta、股息率、主力成本、资金态度、60 日收益、最大回撤 |
| 资金溯源 | 主力净流入、近 5 日主力净流入、超大/大单、主力净占比、换手率、20 日量价成本 |

数据来源原则：

- **不使用模拟行情、不使用随机行情、不手写假数据**。
- 行情/K 线/成交量：来自 AKShare、东方财富公开接口、新浪财经、Yahoo Finance 等真实数据源。
- 实时价格/涨跌幅：优先使用新浪财经或 AKShare，失败再使用可用的真实 fallback。
- 技术指标：基于真实 K 线计算 MA、MACD、RSI、KDJ、BOLL、回撤、波动率。
- 估值/基础资料：来自 AKShare、腾讯行情估值补充、东方财富全量快照等公开数据源。
- 资金/风险/事件：来自资金流、龙虎榜、限售解禁、公告、新闻、研报等公开数据源；如果数据源不可用则显示“暂无/获取失败”，不会编造。
- “主力成本、控盘、托底意愿”等无法直接从公开接口确认的字段，只能用真实的价格、成交量、换手率、资金流等指标做近似推断，并会明确标注为模型推断。
- PEG 优先使用 PE ÷ 一致预期 EPS 增速，若一致预期缺失则兜底使用财务摘要中的净利润同比、归母净利润同比或 EPS 同比；Beta 需要指数收益序列，当前个股页没有足够数据时显示“暂无”，不使用假基准。
- Beta 使用个股真实 K 线与沪深300指数 K 线收益率计算，指数历史行情优先走 AKShare 东财接口，失败时回退新浪指数历史行情。
- 股息率优先读取公开基础资料中的真实字段；若字段缺失，则用最近已实施现金分红/当前价推导，并标注为历史分红按现价推导。
- “相对强弱”当前展示本股 20 日真实 K 线收益，并明确标注“非全市场 RPS”；后续接入全市场收益池后可升级为严格 RPS。

## 从零开始部署运行

本项目优先面向 Windows 新人用户设计，最简单的方式是下载项目后双击 `start.bat`。首次启动会自动创建虚拟环境、安装依赖并打开 Web 页面。

### 1. 安装 Python

1. 打开 [Python 官网](https://www.python.org/downloads/) 下载 Python 3.10 或更高版本。
2. 安装第一页务必勾选 **Add Python to PATH**。
3. 安装完成后，打开 PowerShell 或命令提示符，输入：

```powershell
python --version
```

如果能看到类似 `Python 3.10.x` / `Python 3.11.x` / `Python 3.12.x`，说明 Python 已安装成功。

### 2. 下载本项目

如果你是从 GitHub 页面下载：

1. 打开项目 GitHub 仓库。
2. 点击绿色 **Code** 按钮。
3. 点击 **Download ZIP**。
4. 下载完成后解压，例如解压到桌面或 `D:\stock_analyzer`。
5. 进入解压后的项目文件夹，确认能看到 `start.bat`、`README.md`、`app.py`。

如果你熟悉 Git，也可以用：

```bash
git clone <本项目仓库地址>
cd stock_analyzer
```

Windows 下直接双击项目根目录的：

```text
start.bat
```

首次运行会自动完成：

1. 检查 Python。

### 7. 常见问题

- **双击后窗口一闪而过**：通常是 Python 没装好，重新安装 Python 并勾选 **Add Python to PATH**。
- **依赖安装很慢**：使用上面的清华镜像命令重新安装依赖。

### 9. 智能推荐策略

智能推荐页支持四类策略：

| 策略 | 适合场景 | 核心口径 |
|------|----------|----------|
| 综合推荐 | 日常观察 | 使用原有技术指标评分和推荐池逻辑 |
| 短线 / 长线 | 技术信号筛选 | 按 MACD、RSI、KDJ、BOLL、均线等指标评分 |
| 激进突破型 | 短线/超短线 | 纯量价突破，要求市值、均线多头、20 日新高和放量同时满足 |
| 多因子稳健型 | 多条件候选池 | 硬性过滤后按技术、财务、连涨、主力净流入趋势、15 日内涨停打分 |

**激进突破型规则：**

- 范围：沪深主板 + 创业板；排除科创板、北交所、ST、退市/异常股票。
- 市值：总市值 `<300亿`。
- 技术条件：`MA5 > MA10 > MA20`。
- 突破条件：最新收盘价创近 `20日` 新高。
- 量能条件：当日成交量 > 前 `5日` 平均成交量 × `1.2`。
- 买入提示：突破确认后观察买入，不承诺实时打板。
- 卖出纪律：次日低开超过 `-5%` 止损；跌破前收且 `5` 分钟不反弹退出；高开冲高分批止盈；平开 `30` 分钟走弱退出。

**多因子稳健型规则：**

- 适用范围：沪深主板 + 创业板。
- 排除范围：科创板、北交所、ST、退市/异常股票。
- 硬性过滤：总市值 `<300亿`。
- 硬性过滤：近 `10日` 涨幅不超过 `35%`。
- 硬性过滤：无重大风险事件。
- 核心因子：均线金叉或均线多头，并且放量。
- 核心因子：最新季报净利润同比 `>20%`，或者最新季报未亏损。
- 核心因子：连续上涨 `3日`。
- 核心因子：主力净流入资金趋势 `>=3000万`。
- 核心因子：既往 `15日` 内有涨停。
- 入选规则：核心因子至少 `3/5` 通过，且综合评分 `>=70`。

策略页会展示真实阶段进度：股票池、市值过滤、K 线轻筛、深度检查和完成状态。历史 K 线默认使用本地缓存并按交易日更新；实时行情不参与原始选股，只在 T+1 计划生成后用于计划内股票的盘中入场检查。需要强制更新策略日 K 缓存时点击页面上的“刷新K线缓存”按钮，该按钮不会读取或生成 T+1 推荐计划。

**Alpha 二次评分：**

- 不接 LLM，不需要模型 API；它是规则化、可解释的本地算法。
- 评分依据：策略原始分、均线/趋势信号、量能、主力资金、板块联动、财务质量、策略条件共振、公告/解禁风险、当日涨幅过热风险。
- 默认 `RECOMMEND_RANKER_ENABLED=true`：推荐列表会显示 Alpha 分、等级、理由和扣分项。
- 默认 `RECOMMEND_RANKER_SORT=false`：只补充评分解释，不改变原推荐模块的排序。
- 如需让 Alpha 分参与排序，设置 `RECOMMEND_RANKER_SORT=true` 后重启应用。

### 常用配置

- `RUNTIME_CACHE_DIR`：运行缓存目录，默认 `.cache/`，用于离线行情缓存和主板股票池缓存。
- `API_AUTH_KEY`：API 鉴权密钥；如果 API 服务不只监听本地地址，建议务必设置。
- `CACHE_TTL_FUNDAMENTALS`：A股基础资料/估值缓存时间，默认 3600 秒。
- `CACHE_TTL_STOCK_FINANCIAL`：财务摘要缓存时间，默认 86400 秒。
- `CACHE_TTL_STOCK_RESEARCH`：研报/分红/板块归因缓存时间，默认 86400 秒。
- `CACHE_TTL_STOCK_RISK_EVENTS`：公告/风险事件缓存时间，默认 86400 秒。
- `CACHE_TTL_STOCK_FUND_FLOW`：资金流缓存时间，默认 3600 秒。
- `CACHE_TTL_RECOMMENDATION_RESULTS`：智能推荐结果缓存时间，默认 3600 秒。
- `RECOMMEND_RANKER_ENABLED`：是否给智能推荐结果增加 Alpha 评分与解释，默认 `true`。
- `RECOMMEND_RANKER_SORT`：是否按 Alpha 分对推荐结果二次重排，默认 `false`，即不改变原策略排序。
- `CACHE_TTL_WATCHLIST_SUMMARY`：自选股摘要/推送缓存时间，默认 300 秒。
- `STOCK_DATA_SOURCE`：A股数据源优先级，可选 `auto` / `akshare` / `sina` / `yfinance`。
- `SCHEDULE_TIME`：定时任务执行时间，默认 `15:30`。
- `NOTIFY_CHANNELS`：推送渠道，逗号分隔，可选 `wechat`, `feishu`。
- `SECTOR_PUSH_ENABLED`：定时推送是否包含固定四板块推荐，默认 `true`。
- `SECTOR_PUSH_SHORT_TOP_N` / `SECTOR_PUSH_LONG_TOP_N`：每个板块短线/长线推荐数量，默认 `2` / `1`。
- `DAILY_REPORT_ENABLED`：定时任务是否生成每日报告，默认 `true`。
- `DAILY_REPORT_PUSH_ENABLED`：定时任务是否推送完整 Markdown 日报，默认 `true`。
- `DAILY_REPORT_INCLUDE_RECOMMENDATIONS`：日报是否扫描全市场推荐股，默认 `false`，避免和 T+1 预热重复跑全市场推荐。
- `T1_PLAN_AUTO_ENABLED`：是否在调度器中自动预生成 T+1 推荐计划，默认 `true`。
- `T1_PLAN_SCHEDULE_TIME`：T+1 推荐计划预生成时间，默认 `15:45`。
- `T1_PLAN_STRATEGIES` / `T1_PLAN_SECTORS` / `T1_PLAN_NUM_STOCKS`：T+1 预生成使用的策略列表、板块列表和数量，默认 `短线,长线,多因子稳健型,激进突破型` / `全部,苹果概念,特斯拉概念,电力,算力租赁` / `5`。短线/长线会按所有板块生成；多因子稳健型/激进突破型默认只生成 `全部`，避免每天重复跑过重扫描。旧的 `T1_PLAN_STRATEGY` / `T1_PLAN_SECTOR` 仍兼容。
- `T1_PLAN_PREHEAT_EXTENDED_INFO_MAX_SYMBOLS` / `T1_PLAN_PREHEAT_EXTENDED_INFO_TIMEOUT_SECONDS` / `T1_PLAN_PREHEAT_EXTENDED_INFO_DEEP`：T+1 生成后扩展信息预热的股票数、总耗时和是否启用深层信息，默认 `5` / `20` / `false`，只影响扩展资料预热，不改变选股策略、排名或日 K 指标计算。
- `T1_PLAN_STRATEGY_TIMEOUT_SECONDS`：调度器中单个 T+1 策略预生成的总超时，默认 `300` 秒；超时会释放调度锁并记录失败状态，不会用空结果覆盖已有推荐缓存。
- `T1_PLAN_PUSH_ENABLED`：T+1 预生成完成后是否推送推荐计划，默认 `true`；推送内容来自已生成计划，不重新选股。
- `DAILY_REPORT_DIR`：日报输出目录，默认 `reports/history`。
- `AI_DEBATE_ENABLED`：日报是否启用外部 LLM 多空辩论/风控经理层，默认 `false`；开启前需配置 `AI_API_KEY`。
- `AI_DEBATE_MAX_SYMBOLS`：每次日报最多对前 N 只自选股做 LLM 辩论，默认 `3`，用于控制 Actions 耗时和费用。
- 本机配置可复制 `.env.example` 为 `.env`，填入 `AI_API_KEY`、`AI_BASE_URL`、`AI_MODEL`、`FEISHU_WEBHOOK_URL`、`WECHAT_WEBHOOK_URL` 等私密配置；`.env` 已加入 `.gitignore`，不会提交到 GitHub。

### 分层数据服务

项目已开始按 `a-stock-data` 的接口分层思路拆分数据层，但不会直接照搬外部仓库实现：

- `data/providers/`：外部数据源适配器，目前包含 AKShare 个股基础资料、财务摘要、资金流、东方财富个股新闻、财新数据通市场资讯、腾讯行情估值补充，以及旧行情获取器适配层 `LegacyQuoteProvider`；个股新闻兼容 AKShare + pandas/pyarrow 字符串存储差异。
- `data/services/`：业务接口层，目前提供 `FundamentalDataService.get_stock_profile()`、`QuoteDataService` 和 `StockInfoService`。
- 基础资料兜底：`FundamentalDataService` 会缓存 A 股全量基础资料索引，组合东方财富全量快照、上交所主板/科创板、深交所 A 股、北交所列表，用于补齐行业、上市日期、股本、市值、PB 等缺失字段；北交所存量股票切换 `920` 号段时，会按股票名称匹配现代码资料，避免“报一个补一个”。
- `data/cache.py`：统一 JSON 文件缓存，默认落到 `.cache/`；跨进程写锁会记录 PID 和时间戳，遇到空锁、异常锁、已退出进程锁或过期锁时自动清理，避免残留 `.lock` 阻塞日报和推送。
- `data/health.py`：数据源健康状态登记，后续用于多源 fallback。
- `data/models.py`：标准数据模型，当前包含 `StockProfile`。
- `data/runtime.py`：统一第三方接口超时包装和非关键数据源安全调用。

当前 Web 个股分析页会并行请求基础资料，不阻塞 K 线主数据渲染。行情相关入口（K线、实时行情、分时、批量报价、大盘指数、数据源选择）已先收敛到 `QuoteDataService`，后续可以继续把新浪、腾讯、AKShare 等源拆成独立 provider。
个股页还会以非阻塞方式加载“财务 / 资金 / 新闻”折叠区和“市场快讯 / 催化消息”折叠区；Web 首屏拉取财务、资金流、个股新闻和市场资讯核心层，财务展示报告期、资金展示资金流日期，新闻展示最新新闻时间或“暂无”状态；研报/风险事件/板块归因等深层数据保留给日报和决策委员会使用，避免拖慢首次搜索。

### 每日分析报告

`reports/DailyReportService` 复用现有 `QuoteDataService`、`StockInfoService` 和 `StockRecommender`，按 `daily_stock_analysis` 项目的“每日 Markdown 复盘/决策仪表盘”思路补齐本项目目标能力，但保持无数据库、无额外服务依赖：

- 大盘温度：读取上证、深证、沪深300、北证50 实时摘要。
- 自选股摘要：复用 `watchlist.json` 和现有自选股汇总逻辑。
- 今日推荐：默认取短线推荐前 5，只在 CLI 生成报告时执行。
- 研报层：对自选股中的 A 股补充东财个股研报、PDF 链接和同花顺一致预期 EPS。
- 风险事件层：补充龙虎榜、限售解禁和近 30 日个股公告，风险公告会进入风险警报。
- 板块归因层：补充行业/概念归属、板块涨跌幅和简单题材原因。
- 决策面板：输出决策评分、买卖点、关键价位、风险警报、催化因素、操作检查清单和六 Agent 明细。
- A股决策委员会：日报和个股分析页复用 `decision_committee.py` 的六 Agent 结论；阶段 1 最终版已加入 Agent 权重、置信度、关键位、估值/换手/资金/题材/风险事件/执行风控细分评分；阶段 3 最终版已把日报升级为决策仪表盘；阶段 4 最终版支持可选外部 LLM 多空辩论（多头研究员、空头研究员、风控经理），未配置 `AI_API_KEY` 时自动跳过，不影响 GitHub Actions 和飞书定时推送。
- 导出结果：写入 `reports/history/YYYY-MM-DD.md`，并同步覆盖 `reports/history/latest.md`。

定时推送复用 `scheduler.py`：配置 `NOTIFY_CHANNELS` 和对应 webhook 后运行 `python main.py --schedule`，每天 `SCHEDULE_TIME` 会按“自选股摘要 → 四板块推荐 → 每日完整 Markdown 日报”的顺序执行。四板块固定为算力租赁、电力、苹果概念、特斯拉概念，默认每个板块推送短线 2 只 + 长线 1 只，并同步输出激进突破型/多因子稳健型候选；短线/长线沿用沪深主板口径，激进突破型/多因子稳健型扫描沪深主板 + 创业板，科创板、北交所、ST、退市/异常股票不进入推荐池；热门板块页的行业板块、概念板块、个股涨跌幅榜保留全市场，不做主板过滤；每日完整 Markdown 日报默认不再重复扫描全市场推荐股，T+1 推荐由 `T1_PLAN_SCHEDULE_TIME` 的预热任务提前生成。自选股摘要、四板块推荐股和完整 Markdown 日报都会带“交易计划卡片 + 风控防御看板”，配置页一键测试推送仅用于测试 Webhook 连通性。若本地调度器因断电、退出或异常留下旧 `.cache/scheduler.instance.lock`，需要先清理旧锁并重启 `python main.py --schedule`；缓存写锁残留会由 `data/cache.py` 自动识别并清理。若只想保存日报不推送正文，可设置 `DAILY_REPORT_PUSH_ENABLED=false`。

GitHub Actions 已启用工作日北京时间 `15:30` 定时运行 `.github/workflows/daily_analysis.yml`，默认推送到飞书。配置仓库 Secret `FEISHU_WEBHOOK_URL` 后，不需要本地电脑常开；如需改为企业微信，把仓库 Variable `NOTIFY_CHANNELS` 设为 `wechat` 并配置 Secret `WECHAT_WEBHOOK_URL`；如需两个渠道同时推送，设为 `feishu,wechat` 并配置两个 Webhook。云端日报如需包含自选股，再配置 `STOCK_LIST` Secret（如 `600519,600036`，也支持 `捷顺科技,瑞鹄模具,上海电力`）或高级格式 `WATCHLIST_JSON`。如需云端 LLM 多空辩论，再配置 Secret `AI_API_KEY`，并把仓库 Variables 里的 `AI_DEBATE_ENABLED` 设为 `true`。也可以在 Actions 页面手动点击 `Run workflow` 立即测试。详细步骤见 `docs/FEISHU_GITHUB_ACTIONS.md`。

## 指标说明

### 新股/短历史数据

如果股票上市时间较短（例如新股/次新股），数据源只能返回上市后的交易日数据。此时页面会显示说明性提示，而不是错误告警：

- 长周期指标（如 `MA20` / `MA60` / `RSI24` / `BOLL`）可能暂不完整。
- 建议优先参考分时图、价格走势和短线指标。
- 等交易日数据积累后，长周期指标会自动恢复完整计算。

### MACD
- **金叉**：MACD 线上穿信号线，买入信号
- **死叉**：MACD 线下穿信号线，卖出信号
- **柱状图**：MACD 与信号线的差值，反映动量

### RSI（6/12/24 三周期）
- **>70**：超买区域，可能回调
- **<30**：超卖区域，可能反弹
- **30-70**：正常区间

### KDJ
- **K 线**（快线）反映近期价格变动
- **D 线**（慢线）反映价格趋势
- **J 线** = 3K - 2D，波动最大
- **金叉**：K 上穿 D，买入信号
- **死叉**：K 下穿 D，卖出信号

### 布林带
- **上轨**：压力位
- **中轨**：20 日均线，支撑/压力位
- **下轨**：支撑位
- **带宽扩大**：波动加剧
- **带宽收窄**：可能变盘

## 评分系统

推荐股票的评分基于以下因子（总分 0-100）：

- **80-100**：偏多信号（强）
- **65-79**：偏多信号
- **50-64**：观望
- **35-49**：偏空信号
- **0-34**：偏空信号（强）
