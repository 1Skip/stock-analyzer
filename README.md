# 股票分析系统

个人股票技术分析工具，支持 A股/港股/美股，提供 Web 界面和 CLI 两种使用方式。

## 项目定位与使用边界

本项目的核心定位是 **收盘后/开盘前选股 + 盘中执行检查**，不是盘中实时重新选股或高频交易系统。

- **适合使用的场景**：收盘后复盘、生成次日 T+1 候选计划、开盘前查看已生成推荐、盘中对计划内股票做入场/风控检查。
- **不适合作为实时选股依据**：不用于盘中实时扫描全市场重新选股，不根据实时价格不断新增、删除或重排原始推荐列表，不用于打板抢单、秒级交易或高频决策。
- **实时行情的边界**：实时行情只用于最新价展示、分时图、自选股摘要和计划内股票的盘中执行检查；不合并进原始日 K 线参与个股分析页指标计算，也不参与原始选股条件。
- **核心选股口径**：智能推荐的原始选股以稳定的历史日 K、收盘后数据和公开基础/资金/风险数据为主，优先保证策略口径一致和结果可复核。

## 项目协作规则

- **红线来源** — `agent.md` 是项目红线、架构约定、长期协作规则和 Codex 协作偏好的正式来源，包含需求执行、需求记录、个股分析指标口径、真实数据和安全边界。
- **Codex Skill** — `.codex/skills/stock-analyzer/SKILL.md` 是随仓库提交的 Codex 执行 SOP，用于提醒 AI 在本项目中先查红线、按真实链路验证、区分缓存/调度状态，并按要求执行文档同步流程。
- **分工关系** — Skill 不替代 `agent.md`；`agent.md` 记录项目规则，Skill 把规则压缩成 Codex 工作时的检查清单。

## 2026-05-16 智能推荐策略更新

- **推荐范围** — 智能推荐新增支持“沪深主板 + 创业板”，仍排除科创板、北交所、ST、退市风险和明显异常停牌股票；涨跌排行继续保留个股全市场口径。
- **激进突破型** — 纯量价策略，要求总市值 `<300亿`、`MA5 > MA10 > MA20`、最新收盘价创近 `20日` 新高、当日成交量大于前 `5日` 均量 `1.2` 倍；页面和推荐计划都会展示买入观察与隔日超短卖出纪律，次日低开止损线为 `-5%`。
- **多因子稳健型** — 先硬性过滤总市值 `<300亿`、近 `10日` 涨幅不超过 `35%`、无重大风险事件；再按“均线金叉或均线多头并放量、最新季报净利润同比 `>20%` 或未亏损、连续上涨 `3日`、主力净流入资金趋势 `>=3000万`、既往 `15日` 内有涨停”五个核心因子评分，核心因子至少 `3/5` 且综合评分 `>=70` 才进入候选。
- **多因子轻筛兜底** — 轻筛阶段会先剔除近 `10日` 涨幅超过 `35%` 的短期过热候选；若均线/量能、连续 `3日` 上涨、既往 `15日` 内涨停触及三个 K 线核心因子全部不满足，也会直接判为 K 线核心因子不足，不进入深度检查。
- **速度优化** — 历史 K 线使用本地交易日缓存，资金流短时缓存，财务/公告/研报按天缓存；页面提供“刷新K线缓存”按钮用于手动更新推荐策略日 K 缓存，不是读取或生成 T+1 推荐计划。实时行情只用于计划内入场检查和最新价展示，不参与原始选股。
- **推荐服务层** — 网页智能推荐和 T+1 推荐计划共用 `RecommendationService` 分发短线、激进突破型、多因子稳健型；相同策略/板块/数量的网页结果会在短时缓存内复用，避免刷新或切页后重复跑。
- **缓存拆分与进度** — 扩展信息拆为财务、资金流、研报/板块、风险公告等缓存，并加跨进程写锁；财务和资金流分层缓存支持部分命中，资金流短缓存过期时不会丢弃仍有效的财务长缓存；多因子深度检查会持续显示已深查数量、总数和当前命中数，选股条件与评分规则不变。
- **T+1 计划预热** — 15:45 统一预生成 T+1 推荐计划，短线覆盖苹果概念和特斯拉概念，激进突破型/多因子稳健型覆盖全市场；所有策略只使用真实行情、资金、财务、风险事件等公开数据，缺失项会标注为缺失/接口失败/未通过。
- **主力吸货指标** — 个股分析页和推荐计划新增同花顺公式版“主力吸货 / 风险 / 涨跌”，完全由真实日 K 的 high/low/close 推导；这是公式指标复刻，不读取同花顺插件私有数据，不参与选股、评分或排序。
- **文档同步约定** — 每次需求改动或 bug 修复完成后同步记录到 memory；README 只记录用户明确提出、已经确认或已经完成验证的功能与修复，不把未确认的扩展想法写成正式需求。

## 功能特点

### 技术指标
- **K线图** — Plotly 交互式蜡烛图（Web），Matplotlib 静态图（CLI）
- **MACD** — 趋势判断和买卖时机，含金叉/死叉标注
- **RSI** — 6/12/24 三周期相对强弱指数
- **KDJ** — 随机指标，含金叉/死叉标注
- **BOLL** — 布林带，压力位/支撑位
- **MA** — 个股分析页展示 5/10/20/30 四条均线，指标层仍保留 MA60 供长期风控模块使用
- **主力吸货** — 按同花顺公式系统实现 `主力吸货`、`风险`、`涨跌` 三条输出，基于真实前复权日 K 推导，仅用于展示和解释，不改变推荐策略
- **分时图** — 当日价格走势 + 均价线 + 昨收参考线

### AI 辅助解读（可选）
- 将技术指标快照交给 LLM 翻译为自然语言解释，默认折叠，不参与 A股决策委员会评分
- 自动检测 API Key 类型（OpenAI / Gemini / Claude / DeepSeek / 通义千问 等 10+ 模型）
- 主结论以 A股决策委员会 为准，AI 仅用于解释和补充风险表述

### 市场支持
- **A股** — 同花顺日K → 东方财富/腾讯财经 → 新浪财经 → Yahoo Finance 真实回退；中文名称搜索使用全量 A 股名称索引（5515+ 只，内置兜底 + 24h 本地缓存）
- **港股** — 东方财富港股人气榜 → 新浪财经行情补充 → Yahoo Finance → 新浪财经兜底
- **美股** — Yahoo Finance → 新浪财经
- **基础资料/估值** — A股接入分层数据服务，AKShare 个股资料 + 腾讯行情估值补充，展示行业、上市日期、市值、PE/PB、换手率

### 其他功能
- **涨跌排行** — 展示 A 股/港股/美股个股涨幅榜和跌幅榜；A 股页展示同花顺口径热门概念板块、行业板块、指数板块，并展示板块统计的全部、行业、概念三类；港股热门名单优先取东方财富港股人气榜，行情用真实公开源补齐。
- **推荐股票** — 多因子技术分析评分（0-100），当前页面入口保留短线、激进突破型、多因子稳健型；策略推荐池包含沪深主板 + 创业板，排除科创板、北交所、ST、退市/异常股票
- **股票对比** — 多只股票技术指标 + 走势决策仪表盘，含区间收益、最大回撤、波动率、上涨天数占比、MA 状态和相对强弱
- **回测引擎** — 信号→交易模拟，含止损/止盈/中性区间
- **大盘温度** — 上证/深证/沪深300/北证50 实时跟踪
- **自选股** — 持久化管理，支持 A股/港股/美股，侧边栏 mini 分析面板
- **交易计划卡片** — 面向单只股票输出当前动作、买入观察区、加仓确认位、止损线、压力/止盈位和仓位建议
- **风控防御看板** — 以估值、成长、趋势、安全、资金五个维度做风险/机会评分，辅助判断是否追高、低吸、观望或降仓
- **每日决策仪表盘** — CLI/定时导出 Markdown 日报，汇总大盘、自选股、推荐股、研报、公告、龙虎榜、解禁、板块归因和操作检查清单
- **A股决策委员会** — 借鉴 TradingAgents 多角色框架，阶段 2 最终版已完成：六 Agent 覆盖技术分析、资金情绪、基本面、题材板块、风险事件和执行风控，个股页以决策仪表盘展示评分、置信度、仓位、关键价位、买卖点、风险警报、催化因素和 Agent 折叠明细；看多依据/看空因素按单条证据方向归集，避免“空头趋势”“超买”等负向信号误入看多依据
- **决策委员会状态卡片** — 侧边栏固定展示阶段 1-5、本地日报、T+1 预热和 LLM 多空辩论开关，网页端可直接确认本地能力是否启用
- **定时调度** — 收盘后自动生成日报和预热 T+1 计划（默认关闭）
- **三种配色** — A股传统（红涨绿跌）/ 国际惯例（绿涨红跌）/ 色盲友好（蓝涨橙跌）
- **离线缓存** — 所有在线源失败时使用 24h 缓存兜底

### 当前推荐与本地调度规则

- 智能推荐主要用于收盘后/开盘前生成候选股和 T+1 计划；盘中只对计划内股票做执行检查，不做全市场实时重新选股。
- 涨跌排行页用于观察全市场个股涨跌幅：个股涨幅榜/跌幅榜保留创业板、科创板、北交所，不做主板过滤；A 股热门板块展示概念、行业、指数三类，板块统计展示全部、行业、概念三类，并按同花顺官方行业/概念名单过滤。
- 智能推荐和定时推荐计划用于辅助决策：短线策略保留原有沪深主板口径；激进突破型/多因子稳健型扫描沪深主板 + 创业板，科创板、北交所、ST、退市/异常股票不进入推荐池。
- 实时行情只用于最新价展示、分时图、自选股摘要和 T+1 计划内股票的入场检查；不参与原始选股、不改变排序、不新增或删除推荐股票。
- 智能推荐页面会复用最近一次相同“策略 + 板块 + 数量”的推荐结果缓存；首次全市场多因子查询仍按完整条件扫描，不截断候选，不降低覆盖率。
- 智能推荐页面的买卖点在推荐计划生成后追加，使用收盘后的日 K、BOLL、均线、支撑/压力和风险字段生成；不做盘中实时追高、跌破支撑或暂停入场重算。
- 15:30 定时任务只负责自选股摘要 + 大盘/完整 Markdown 日报，不再生成固定四板块推荐。
- 15:45 定时任务统一预生成 T+1 推荐计划：短线覆盖苹果概念、特斯拉概念；激进突破型/多因子稳健型只生成全市场 `全部`，默认每组 5 只。
- 自选股摘要、T+1 推荐计划和每日完整日报都会输出“交易计划卡片 + 风控防御看板 + 资金博弈溯源”；自选股摘要和 T+1 推荐计划还会展示主力吸货公式指标，与网页端个股分析口径保持一致。

### 交易计划与风控防御看板

个股分析页已把原有“买卖点与仓位”升级成更明确的单股交易计划模块：

- **交易计划卡片**：回答“当前能不能买、买在哪里、错了在哪里止损、涨到哪里先减仓”。
- **交易状态机**：把单股状态明确落到“防御观望 / 底部观察 / 试探建仓 / 趋势持有 / 压力减仓 / 风险回避”，并给出下一步触发条件。
- **风控防御看板**：回答“这只股票现在是安全低吸、趋势持有、追高风险，还是应该回避”。
- **资金博弈溯源**：展示当日主力、近 5 日主力、超大/大单、主力净占比、换手率、20 日量价成本等字段，区分公开数据与模型推断。

计划展示字段：

| 模块 | 展示内容 |
|------|----------|
| 当前动作 | 建仓 / 轻仓试探 / 持有 / 等待确认 / 降仓回避 |
| 买入观察区 | 接近支撑位、布林下轨、MA20/MA60 或放量突破确认位 |
| 加仓确认位 | 重新站上关键均线、突破压力位、趋势和资金共振 |
| 止损线 | 跌破支撑、跌破布林下轨、跌破 MA60 或风险事件触发 |
| 止盈/减仓位 | 接近压力位、偏离均线过大、RSI 超买或放量滞涨 |
| 仓位建议 | 观察仓 / 1-2 成 / 2-3 成 / 降仓 / 空仓 |
| 风控评分 | 估值、成长、趋势、安全、资金五维评分和解释 |
| 状态机 | 防御观望、底部观察、试探建仓、趋势持有、压力减仓、风险回避 |
| 核心指标 | PEG、相对强弱、Beta、股息率、主力成本、资金态度、60 日收益、最大回撤；有真实值或可推导值才进入主看板 |
| 资金溯源 | 主力净流入、近 5 日主力净流入、超大/大单、主力净占比、换手率、20 日量价成本；仅展示本次有可用数值的行 |

数据来源原则：

- **不使用模拟行情、不使用随机行情、不手写假数据**。
- 行情/K 线/成交量：来自同花顺、AKShare、东方财富/腾讯财经公开接口、新浪财经、Yahoo Finance 等真实数据源。
- 实时价格/涨跌幅：优先使用新浪财经或 AKShare，失败再使用可用的真实 fallback。
- 技术指标：基于真实 K 线计算 MA、MACD、RSI、KDJ、BOLL、回撤、波动率。
- 估值/基础资料：来自 AKShare、腾讯行情估值补充、东方财富全量快照等公开数据源。
- 资金/风险/事件：来自资金流、龙虎榜、限售解禁、公告、新闻、研报等公开数据源；如果数据源不可用则显示“暂无/获取失败”，不会编造。
- “主力成本、控盘、托底意愿”等无法直接从公开接口确认的字段，只能用真实的价格、成交量、换手率、资金流等指标做近似推断，并会明确标注为模型推断。
- PEG 优先使用 PE ÷ 一致预期 EPS 增速，若一致预期缺失则兜底使用财务摘要中的净利润同比、归母净利润同比或 EPS 同比；Beta 需要指数收益序列，当前个股页没有足够数据时显示“暂无”，不使用假基准。
- Beta 使用个股真实 K 线与沪深300指数 K 线收益率计算，指数历史行情优先走 AKShare 东财接口，失败时回退新浪指数历史行情。
- 股息率优先读取公开基础资料中的真实字段；若字段缺失，则用最近已实施现金分红/当前价推导，并标注为历史分红按现价推导。
- “相对强弱”当前展示本股 20 日真实 K 线收益，并明确标注“非全市场 RPS”；后续接入全市场收益池后可升级为严格 RPS。
- 风控防御看板遵循“有值展示、无值降噪”的展示原则：PEG、股息率、资金态度、Beta 等可选指标在当前股票本次接口未返回可用字段时，不在主指标区反复显示“暂无/缺失”；接口失败、源无数据或字段不足会保留在“数据质量 / 风险提示”里，便于排查但不干扰主结论。
- 资金态度和主力净流入不是固定隐藏字段；当东方财富等公开资金流源返回主力净流入、主力净占比或近 5 日主力净流入时，页面继续展示资金博弈溯源和资金态度。不同股票或不同时段接口失败时，只隐藏无值行并记录数据缺口。

## 从零开始部署运行

本项目支持 Windows、macOS 和 Linux 本地运行。Windows 用户可以直接双击 `start.bat`；macOS / Linux 用户可以运行 `start.sh`。首次启动会自动创建虚拟环境、安装依赖并打开 Web 页面。

### 1. 安装 Python

1. 打开 [Python 官网](https://www.python.org/downloads/) 下载 Python 3.10 或更高版本。
2. Windows 安装第一页务必勾选 **Add Python to PATH**；macOS 安装官方包或 Homebrew 版本均可。
3. 安装完成后，打开 PowerShell、命令提示符或终端，输入：

```powershell
python --version
```

macOS / Linux 如果 `python` 不可用，使用：

```bash
python3 --version
```

如果能看到类似 `Python 3.10.x` / `Python 3.11.x` / `Python 3.12.x`，说明 Python 已安装成功。

### 2. 下载本项目

如果你是从 GitHub 页面下载：

1. 打开项目 GitHub 仓库。
2. 点击绿色 **Code** 按钮。
3. 点击 **Download ZIP**。
4. 下载完成后解压，例如解压到桌面或 `D:\stock_analyzer`。
5. 进入解压后的项目文件夹，确认能看到 `start.bat`、`start.sh`、`README.md`、`app.py`。

如果你熟悉 Git，也可以用：

```bash
git clone <本项目仓库地址>
cd stock_analyzer
```

### 3. 一键启动 Web 系统

Windows 下直接双击项目根目录的：

```text
start.bat
```

macOS / Linux 下打开终端进入项目目录，首次运行先给脚本执行权限：

```bash
chmod +x start.sh
./start.sh
```

首次运行会自动完成：

1. 检查 Python。
2. 创建 `.venv` 虚拟环境。
3. 安装 `requirements.txt` 依赖。
4. 启动 Streamlit Web 服务。
5. 自动打开浏览器访问 `http://localhost:8501`。

以后再次使用，Windows 仍然双击 `start.bat`；macOS / Linux 在项目目录运行 `./start.sh` 即可。

### 4. 开始使用

浏览器打开后，可以按这个顺序体验：

1. 进入 **个股分析**，输入股票代码或中文名，例如 `002609`、`捷顺科技`、`贵州茅台`。
2. 查看 **A股决策委员会**：综合评分、仓位、买卖点、支撑/压力、风险警报。
3. 点击 **加入自选**，以后可以在左侧自选股侧栏快速打开。
4. 进入 **股票对比**，输入多只股票做横向比较。
5. 进入 **回测验证**，用历史数据验证技术信号。
6. 如需每日复盘，可开启本地定时任务生成日报。

### 7. 常见问题

- **双击后窗口一闪而过**：通常是 Python 没装好，重新安装 Python 并勾选 **Add Python to PATH**。
- **macOS 提示没有权限运行 `start.sh`**：在项目目录执行 `chmod +x start.sh` 后再运行 `./start.sh`。
- **依赖安装很慢**：使用上面的清华镜像命令重新安装依赖。
- **浏览器没有自动打开**：手动打开 `http://localhost:8501`。
- **端口被占用**：关闭旧的命令行窗口，或在任务管理器结束旧的 Python/Streamlit 进程后重新双击 `start.bat`。
- **输入中文股票名查不到**：优先尝试 6 位股票代码；若仍失败，可以删除 `.cache/stock_name_index.json` 后重启，让项目重新构建名称索引。

### 8. 开启 AI 解读（可选）

普通行情、技术指标、推荐计划和回测不需要模型 API。只有使用 **个股分析 -> AI 辅助解读**，或开启日报里的外部 LLM 多空辩论时，才需要配置模型 Key。

配置步骤：

1. 到你选择的模型服务商控制台创建 API Key，例如 DeepSeek、OpenAI、通义千问、Claude、Gemini 等。
2. 在项目根目录复制 `.env.example`，重命名为 `.env`。
3. 打开 `.env`，填写下面几项：

```text
AI_ENABLED=true
AI_API_KEY=你的模型API_KEY
AI_BASE_URL=https://api.deepseek.com
AI_MODEL=deepseek-chat
```

如果使用其他模型服务商，请把 `AI_BASE_URL` 和 `AI_MODEL` 改成该服务商文档里的地址和模型名。常见例子：

| 服务商 | `AI_BASE_URL` 示例 | `AI_MODEL` 示例 |
|--------|--------------------|-----------------|
| DeepSeek | `https://api.deepseek.com` | `deepseek-chat` |
| OpenAI 兼容服务 | 服务商提供的 OpenAI-compatible Base URL | 服务商提供的模型名 |

`.env` 是本机私密文件，已经加入 `.gitignore`，不要发给别人，也不要提交到 GitHub。未配置 API Key 时，AI 解读会不可用或自动跳过，但不影响普通股票分析。

### 9. 股票输入方式

个股分析页支持**股票代码/中文名称搜索**（如 `平安银行`、`报喜鸟`、`茅台`、`捷顺科技`），输入后按 **Enter 键**直接分析。A股名称索引优先使用 24h 本地缓存 `.cache/stock_name_index.json`，并内置一份全量兜底索引，GitHub/离线环境也能识别常见 A 股中文名称。

### 10. 智能推荐策略

智能推荐页支持四类策略：

| 策略 | 适合场景 | 核心口径 |
|------|----------|----------|
| 综合推荐 | 日常观察 | 使用原有技术指标评分和推荐池逻辑 |
| 短线 | 短线综合候选 | 技术指标打底，叠加基本面/估值、财报盈利、资金流、消息面、热门板块和风险公告评分 |
| 激进突破型 | 短线/超短线 | 纯量价突破，要求市值、均线多头、20 日新高和放量同时满足 |
| 多因子稳健型 | 多条件候选池 | 硬性过滤后按技术、财务、连涨、主力净流入趋势、15 日内涨停打分 |

**激进突破型规则：**

- 范围：沪深主板 + 创业板；排除科创板、北交所、ST、退市/异常股票。
- 市值：总市值 `<300亿`。
- 技术条件：`MA5 > MA10 > MA20`。
- 突破条件：最新收盘价创近 `20日` 新高。
- 量能条件：当日成交量 > 前 `5日` 平均成交量 × `1.2`。
- 买入提示：突破确认后观察买入，不承诺实时打板。
- 卖出纪律：次日低开超过 `-5%` 止损；跌破前收且 `5` 分钟不反弹退出；高开冲高分批止盈；平开 `30` 分钟走弱退出。

**多因子稳健型规则：**

- 适用范围：沪深主板 + 创业板。
- 排除范围：科创板、北交所、ST、退市/异常股票。
- 硬性过滤：总市值 `<300亿`。
- 硬性过滤：近 `10日` 涨幅不超过 `35%`。
- 硬性过滤：无重大风险事件。
- 核心因子：均线金叉或均线多头，并且放量。
- 核心因子：最新季报净利润同比 `>20%`，或者最新季报未亏损。
- 核心因子：连续上涨 `3日`。
- 核心因子：主力净流入资金趋势 `>=3000万`。
- 核心因子：既往 `15日` 内有涨停。
- 入选规则：核心因子至少 `3/5` 通过，且综合评分 `>=70`。

策略页会展示真实阶段进度：股票池、市值过滤、K 线轻筛、深度检查和完成状态。历史 K 线默认使用本地缓存并按交易日更新；实时行情不参与原始选股，只在 T+1 计划生成后用于计划内股票的盘中入场检查。需要强制更新策略日 K 缓存时点击页面上的“刷新K线缓存”按钮，该按钮不会读取或生成 T+1 推荐计划。

**Alpha 二次评分：**

- 不接 LLM，不需要模型 API；它是规则化、可解释的本地算法。
- 评分依据：策略原始分、均线/趋势信号、量能、主力资金、板块联动、财务质量、策略条件共振、公告/解禁风险、当日涨幅过热风险。
- 默认 `RECOMMEND_RANKER_ENABLED=true`：推荐列表会显示 Alpha 分、等级、理由和扣分项。
- 默认 `RECOMMEND_RANKER_SORT=false`：只补充评分解释，不改变原推荐模块的排序。
- 如需让 Alpha 分参与排序，设置 `RECOMMEND_RANKER_SORT=true` 后重启应用。

**短线闭环学习实验版：**

- 仅作用于短线策略，激进突破型和多因子稳健型不接入学习层，原规则保持不变。
- 股票池继续使用沪深主板，学习层额外防止科创板代码误入；不使用写死的 `score >= 65` 作为动态门槛。
- 动态评分门槛来自已保存短线 T+1 计划的真实后续 K 线回看，按分数段统计 1 日收益和胜率；完成样本不足时只展示“样本不足”，不启用动态门槛。
- 样本足够且分数段表现优于基准时，短线候选会写入 `learning_bonus`、`learned_alpha_score` 和学习原因；学习层不改短线原始技术评分，不改激进型/稳健型规则。

### 常用配置

- `CACHE_TTL_STOCK_FUND_FLOW`：资金流缓存时间，默认 3600 秒。

- `CACHE_TTL_WATCHLIST_SUMMARY`：自选股摘要缓存时间，默认 300 秒。
- `STOCK_DATA_SOURCE`：A股数据源优先级，可选 `auto` / `akshare` / `sina` / `yfinance`。
- `WENCAI_COOKIE` / `IWENCAI_COOKIE`：可选，同花顺问财 Cookie。热门行业/概念/指数板块优先走同花顺热榜；问财用于可用时补充板块统计和兜底查询，未配置或失效时会继续使用同花顺页面、东方财富、新浪行业和本地缓存等真实来源。

- `DAILY_REPORT_ENABLED`：定时任务是否生成每日报告，默认 `true`。

- `T1_PLAN_SCHEDULE_TIME`：T+1 推荐计划预生成时间，默认 `15:45`。
- `T1_PLAN_STRATEGIES` / `T1_PLAN_SECTORS` / `T1_PLAN_NUM_STOCKS`：T+1 预生成使用的策略列表、板块列表和数量，默认 `短线,多因子稳健型,激进突破型` / `苹果概念,特斯拉概念` / `5`。短线会按苹果概念、特斯拉概念生成；多因子稳健型/激进突破型默认只生成 `全部`，避免重复跑过重扫描。旧的 `T1_PLAN_STRATEGY` / `T1_PLAN_SECTOR` 仍兼容。
- `T1_PLAN_PREHEAT_EXTENDED_INFO_MAX_SYMBOLS` / `T1_PLAN_PREHEAT_EXTENDED_INFO_TIMEOUT_SECONDS` / `T1_PLAN_PREHEAT_EXTENDED_INFO_DEEP`：T+1 生成后扩展信息预热的股票数、总耗时和是否启用深层信息，默认 `5` / `20` / `false`，只影响扩展资料预热，不改变选股策略、排名或日 K 指标计算。
- `T1_PLAN_STRATEGY_TIMEOUT_SECONDS`：调度器中单个 T+1 策略预生成的总超时，默认 `300` 秒；超时会释放调度锁并记录失败状态，不会用空结果覆盖已有推荐缓存。
- `DAILY_REPORT_DIR`：日报输出目录，默认 `reports/history`。
- `AI_DEBATE_ENABLED`：日报是否启用外部 LLM 多空辩论/风控经理层，默认 `false`；开启前需配置 `AI_API_KEY`。
- `AI_DEBATE_MAX_SYMBOLS`：每次日报最多对前 N 只自选股做 LLM 辩论，默认 `3`，用于控制日报生成耗时和费用。
- 本机配置可复制 `.env.example` 为 `.env`，填入 `AI_API_KEY`、`AI_BASE_URL`、`AI_MODEL` 等私密配置；`.env` 已加入 `.gitignore`，不会提交到 GitHub。

### 分层数据服务

项目已开始按 `a-stock-data` 的接口分层思路拆分数据层，但不会直接照搬外部仓库实现：

- `data/providers/`：外部数据源适配器，目前包含 AKShare 个股基础资料/财务/资金/新闻、同花顺日 K、东方财富日 K/实时/分时/指数、腾讯日 K/实时、Yahoo K线/报价、新浪日 K/实时/分时，以及旧行情获取器适配层 `LegacyQuoteProvider`；个股新闻兼容 AKShare + pandas/pyarrow 字符串存储差异。
- `data/services/`：业务接口层，目前提供 `FundamentalDataService.get_stock_profile()`、`QuoteDataService` 和 `StockInfoService`。
- 基础资料兜底：`FundamentalDataService` 会缓存 A 股全量基础资料索引，组合东方财富全量快照、上交所主板/科创板、深交所 A 股、北交所列表，用于补齐行业、上市日期、股本、市值、PB 等缺失字段；北交所存量股票切换 `920` 号段时，会按股票名称匹配现代码资料，避免“报一个补一个”。
- `data/cache.py`：统一 JSON 文件缓存，默认落到 `.cache/`；跨进程写锁会记录 PID 和时间戳，遇到空锁、异常锁、已退出进程锁或过期锁时自动清理，避免残留 `.lock` 阻塞日报。
- `data/health.py`：数据源健康状态登记，后续用于多源 fallback。
- `data/models.py`：标准数据模型，当前包含 `StockProfile`。
- `data/runtime.py`：统一第三方接口超时包装和非关键数据源安全调用。

当前 Web 个股分析页会并行请求基础资料，不阻塞 K 线主数据渲染。行情相关入口（K线、实时行情、分时、批量报价、大盘指数、数据源选择）已收敛到 `QuoteDataService`，具体源细节继续下沉到独立 provider，方便针对单一数据源做字段契约测试、超时兜底和故障诊断。
个股分析页 A 股日 K 固定在线优先：先请求 `1y` 前复权日 K，只有在线结果异常或为空时才读取本地真实 K 线缓存兜底；兜底提示“在线日K源暂不可用”只应在实际走本地缓存时出现。成交量图和量能指标不会使用缺少日期的实时成交量覆盖最后一根日 K；离线缓存会保留或推断 `volume_unit`，避免本地缓存兜底后股/手单位混用。
个股页还会以非阻塞方式加载“财务 / 资金 / 新闻”折叠区和“市场快讯 / 催化消息”折叠区；Web 首屏拉取财务、资金流、个股新闻和市场资讯核心层，财务展示报告期、资金展示资金流日期，新闻展示最新新闻时间或“暂无”状态；研报覆盖和 EPS 一致预期会在“财务 / 资金 / 新闻”区显式展示为辅助信息，有研报时仍可作为催化提示，但不参与核心决策分、置信度、动作和仓位。

### 每日分析报告

`reports/DailyReportService` 复用现有 `QuoteDataService`、`StockInfoService` 和 `StockRecommender`，按 `daily_stock_analysis` 项目的“每日 Markdown 复盘/决策仪表盘”思路补齐本项目目标能力，但保持无数据库、无额外服务依赖：

- 大盘温度：读取上证、深证、沪深300、北证50 实时摘要。
- 自选股摘要：复用 `watchlist.json` 和现有自选股汇总逻辑。
- 今日推荐：默认取短线推荐前 5，只在 CLI 生成报告时执行。
- 研报层：对自选股中的 A 股补充东财个股研报、PDF 链接和同花顺一致预期 EPS。
- 风险事件层：补充龙虎榜、限售解禁和近 30 日个股公告，风险公告会进入风险警报。
- 板块归因层：补充行业/概念归属、板块涨跌幅和简单题材原因。
- 决策面板：输出决策评分、买卖点、关键价位、风险警报、催化因素、操作检查清单和六 Agent 明细；看多依据/看空因素按单条证据方向归集，负向技术信号不会误放入看多依据。
- A股决策委员会：日报和个股分析页复用 `decision_committee.py` 的六 Agent 结论；阶段 1 最终版已加入 Agent 权重、置信度、关键位、估值/换手/资金/题材/风险事件/执行风控细分评分；阶段 3 最终版已把日报升级为决策仪表盘；阶段 4 最终版支持可选外部 LLM 多空辩论（多头研究员、空头研究员、风控经理），未配置 `AI_API_KEY` 时自动跳过，不影响本地日报和定时预热。
- 导出结果：写入 `reports/history/YYYY-MM-DD.md`，并同步覆盖 `reports/history/latest.md`。

定时调度复用 `scheduler.py`：运行 `python main.py --schedule` 后，每天 `SCHEDULE_TIME` 默认 15:30 执行自选股摘要 + 每日完整 Markdown 日报；`T1_PLAN_SCHEDULE_TIME` 默认 15:45 执行统一 T+1 推荐计划预热。T+1 计划中，短线覆盖苹果概念、特斯拉概念，激进突破型/多因子稳健型只生成 `全部`；短线沿用沪深主板口径，激进突破型/多因子稳健型扫描沪深主板 + 创业板，科创板、北交所、ST、退市/异常股票不进入推荐池。涨跌排行页的个股涨幅榜/跌幅榜保留全市场，不做主板过滤。若本地调度器因断电、退出或异常留下旧 `.cache/scheduler.instance.lock`，需要先清理旧锁并重启 `python main.py --schedule`；缓存写锁残留会由 `data/cache.py` 自动识别并清理。


## 指标说明

### 新股/短历史数据

如果股票上市时间较短（例如新股/次新股），数据源只能返回上市后的交易日数据。此时页面会显示说明性提示，而不是错误告警：

- 长周期指标（如 `MA20` / `MA60` / `RSI24` / `BOLL`）可能暂不完整。
- 建议优先参考分时图、价格走势和短线指标。
- 等交易日数据积累后，长周期指标会自动恢复完整计算。

### MACD
- **金叉**：MACD 线上穿信号线，买入信号
- **死叉**：MACD 线下穿信号线，卖出信号
- **柱状图**：MACD 与信号线的差值，反映动量

### RSI（6/12/24 三周期）
- **>70**：超买区域，可能回调
- **<30**：超卖区域，可能反弹
- **30-70**：正常区间

### KDJ
- **K 线**（快线）反映近期价格变动
- **D 线**（慢线）反映价格趋势
- **J 线** = 3K - 2D，波动最大
- **金叉**：K 上穿 D，买入信号
- **死叉**：K 下穿 D，卖出信号

### 布林带
- **上轨**：压力位
- **中轨**：20 日均线，支撑/压力位
- **下轨**：支撑位
- **带宽扩大**：波动加剧
- **带宽收窄**：可能变盘

## 评分系统

推荐股票的评分基于以下因子（总分 0-100）：

- **80-100**：偏多信号（强）
- **65-79**：偏多信号
- **50-64**：观望
- **35-49**：偏空信号
